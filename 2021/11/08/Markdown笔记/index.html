写在前面的话

在过去的大多数时候，无论在工作中或者生活中，我总是想着是要记录一下在上述的两种场景中遇到的麻烦以及解决这些麻烦的方法，但苦于没有趁手的记录工具，导致要么想法搁浅，要么就是记录过于分散没有办法集中管理，实在是让人头疼不已。

Markdown 让一切变得简单起来，至少对我来说是这样，当然要用好它其实还是有一定的门槛的，但相信我，这绝对是一劳永逸的事情，非常值得。学习它的方式有很多种，为了高效快速的掌握它，我直接按 官方文档学习。

什么是Markdown？

官方的解释如下:

Markdown is a lightweight markup language that you can use to add formatting elements to plaintext text documents.

轻量级标记语言：一种将文本Text以及文本相关的其他信息(文本结构和表示信息)结合起来，展现出关于文档结构和数据结构处理细节的计算机文字编码,结合方式就是使用markup进行标记。类似的有HTML，XML等标记语言。

为什么要用markdown？

窃以为markdown是一种记录文字和代码的方式，比以往任何方式都更简洁高效，在这个网络异常发达的时代，任何一个渴望进步以及始终保持学习习惯的人都需要在网络上记录自己的学习和思考，而markdown就是面向Web面向网络的，甚至可以说是面向未来的记录方式。

新时代下任何一个独立的人都没有理由不掌握。

块元素 Block Elements

句子和换行符 Paragraph and line breaks

一个段落只是一个或多个连续的文本行。 在 Markdown中，段落由两个或多个空行分隔，或者在行某用 <br/>。支持Html的时候直接用标签br，反之可以使用两个或更多空格结束一行，然后键入return。大多数其他markdown解析器都会忽略单行符，所以为了让其他markdown解析器识别你的换行符，所以要在行尾留两个空格。

注意:除非段落在列表中，否则不要用空格或制表符缩进段落。

标题 Headers

标题在行首使用 1-6 个散列 (#) 字符，对应于标题级别 1-6。

# This is an H1

## This is an H2

###### This is an H6

块引用 Blockquotes

要创建块引用，请在段落前面添加>。
块引号可以包含多个段落。在段落之间的空行上加上一个>。

This is a blockquote with two paragraphs. This is first paragraph.

This is second pragraph. Vestibulum enim wisi, viverra nec, fringilla in, laoreet vitae, risus.

This is another blockquote with one paragraph.

列表 Lists

无序列表 un-ordered list

有序列表 ordered list

要在列表中添加另一个元素，同时保持列表的连续性，请将元素缩进四个空格或一个Tab，之后键入其他元素如:

  • This is the first list item.

  • Here’s the second list item.

    A blockquote would look great below the second list item.

  • And here’s the third list item.

  • This is the first list item.

  • Here’s the second list item.

    A blockquote would look great below the second list item.

  • And here’s the third list item.

水平线 Horizontal Rules

目录 (TOC)(typra 支持)

输入[toc]并按Return键。 这将创建一个“目录”部分。 TOC 从文档中提取所有标题，并在您添加到文档时自动更新其内容。
